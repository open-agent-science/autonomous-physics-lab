template CSV files with real extraction artifacts.

```

        Preferred locator: https://doi.org/10.1103/PhysRevB.53.16338
        If the copy is local-only, record the maintainer-held path outside
        the repository without committing the file.

        Required fields: retrieved_from, retrieved_at_utc, retrieved_by,
        access_method, version_of_record_status, and any access notes.

[ ] 3.  Compute SHA-256 for the source copy.
        Cross-platform command:
        python3 -c "import hashlib, pathlib, sys; p=pathlib.Path(sys.argv[1]); print(hashlib.sha256(p.read_bytes()).hexdigest())" <source-file>

[ ] 4.  Decide redistribution posture before committing anything.
        Default: metadata_only. Do not commit the APS PDF, full figure image,
        cropped panel, or rasterized screenshot unless a maintainer records
        file_committed_with_permission.

[ ] 5.  Confirm the exact figure and panel.
        Record the figure number, panel label if any, axis labels, units,
        plotted quantity, and whether the x-axis is radius_nm or diameter_nm.

[ ] 6.  Confirm source semantics.
        Record whether plotted points are direct spectroscopic measurements,
        assignment-derived/model-only values, prior-source points, or mixed.
        Non-direct or prior-source points must be excluded unless separately
        registered and traceable.

[ ] 7.  Confirm uncertainty semantics.
        Record whether uncertainty is per point, per method, absent in source,
        or inferred only from digitization read error. Absence is not a reason
        to silently include points.

[ ] 8.  Confirm digitization tool.
        Record tool name, version, calibration method, and export format.
        WebPlotDigitizer or an equivalent axis-calibrated tool is required.

[ ] 9.  Preserve reviewer replay.
        A second curator must be able to re-open the same source copy, inspect
        the same panel, apply the same calibration anchors, and reproduce the
        extracted point states within declared tolerance.

[ ] 10. Keep TASK-0225 blocked.
        This handoff and any source copy alone do not unblock the
        measurement-versus-model benchmark. Direct rows must be curated and
        then pass the readiness gate first.
```

## Local-Only Handling Rules

If the maintainer supplies a PDF or figure copy but redistribution is not
cleared, keep the source copy outside git and commit only metadata derived from
the copy:

- source locator and DOI;
- retrieval/access metadata;
- checksum;
- figure/panel identifier;
- axis calibration anchors;
- extracted numeric coordinates generated by the digitization tool;
- inclusion/exclusion states and uncertainty notes.

Do not commit the publisher PDF, article pages, full tables, raw figures,
cropped panels, or screenshots under the default `metadata_only` posture.

## Exact Artifact Names For The Later Digitization Task

The existing template package lives under:

```text

```

A future approved tool run should replace templates with these files:

```text

extracted_points.csv
extraction_ledger.csv
notes.md
```

Required content:

- `axis_calibration.csv`: axis name, source unit, plot pixel/coordinate value,
  calibrated value, anchor role, and calibration note.
- `extracted_points.csv`: point id, raw x/y coordinates, calibrated size,
  calibrated energy, material, property kind, figure ref, and uncertainty
  fields.
- `extraction_ledger.csv`: point id, inclusion state, exclusion reason when
  applicable, primary-source attribution, model/direct classification, reviewer
  notes, and replay status.
- `notes.md`: source-copy metadata, checksum, redistribution posture, tool
  name/version, target panel, calibration summary, uncertainty policy, and
  reviewer replay instructions.

The template files
`axis_calibration_template.csv`, `extracted_points_template.csv`, and
`extraction_ledger_template.csv` must remain templates until the legitimate
source copy and deterministic tool run exist.
