## Scientific Core Versus Packaging

The contested replay did not contest the dimensional validator's core numerical
output:

- 74/74 curated labels agreed with the deterministic validator.
- `agreement_fraction` remained `1.0`.
- `best_verdict` remained `VALID`.
- Reproduced numeric drift was `0.0`.

The contest is packaging/provenance integrity: three named checks in the
published verification block are useful Gate A publication statements, but they
are not regenerated by RESULT-0020's recorded command.

This distinction matters for `CLAIM-0005`: the claim should not cite
RESULT-0020 as independently validated until the maintainer resolves whether
Gate B can apply to the reproducible core alone or requires a repackaged
artifact.

## Maintainer Decision Options

### Option A: Validate The Reproducible Core Only

Decision: declare that Gate B may compare only the four replay-emitted checks
plus core metrics, while the three extra checks remain labeled Gate A packaging
annotations.

Consequence:

- RESULT-0020 could become eligible for `AGENT_VALIDATED` after a follow-up
  replay that explicitly ignores or relabels the packaging-only checks.
- The scientific statement remains narrow: dimensional-consistency
  classification over the frozen 74-item fixture, not empirical correctness.
- `CLAIM-0005` should remain `DRAFT` until a separate Gate C decision, but its
  evidence table could later reference RESULT-0020 as validated-in-core.

Implementation shape:

- Add or adjust comparison policy so Gate B excludes explicitly labeled
  `gate_a_packaging` checks.
- Do not change RESULT-0020 metrics.
- Re-run the Gate B helper and record the policy in a review note.

### Option B: Keep RESULT-0020 At AGENT_PUBLISHED With A Blocker

Decision: preserve the current contested state until the artifact is
repackaged, with no tier upgrade.

Consequence:

- No repository mutation is required beyond this brief.
- `CLAIM-0005` remains protected from relying on contested validation.
- The core metrics can still be cited as agent-published evidence with the
  packaging blocker attached.

Implementation shape:

- Close TASK-0782 as maintainer-blocked or keep it blocked with this brief as
  the decision packet.
- Do not alter the Gate B comparator.

### Option C: Require A Deterministic Packaging Re-Emission

Decision: require a packaging helper or workflow mode to deterministically emit
the three Gate A checks before any Gate B pass is allowed.

Consequence:

- Highest provenance strictness.
- RESULT-0020 stays `AGENT_PUBLISHED` until the publication packaging is
  regenerated by code.
- `CLAIM-0005` remains `DRAFT` and should not be strengthened from RESULT-0020.

Implementation shape:

- Add a deterministic packaging command that emits
  `zero_disagreement_ledger`, `frozen_input_checksum`, and
  `protected_result_not_rewritten`.
- Re-run Gate B against the repackaged artifact.
- Keep the dimensional metrics and verdict unchanged.

## Recommended Path

Prefer **Option A** if the maintainer wants the fastest scientifically fair
resolution: it preserves the reproducible dimensional core while making the
Gate A packaging boundary explicit. Prefer **Option C** if the maintainer wants
Gate B to require full end-to-end reproduction of every committed verification
check. **Option B** is safest if maintainer time is unavailable.
