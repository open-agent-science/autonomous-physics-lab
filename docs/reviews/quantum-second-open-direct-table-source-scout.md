# Quantum Second Open Direct-Table Source Scout

**Task:** `TASK-0656`

**Mode:** planning only (single-candidate source scout)

**Verdict:** `PROMISING_NEW_FAMILY_DIRECT_TABLE_UNVERIFIED` — preserve blocker

## Scope

This scout checks **exactly one** additional candidate source family beyond the
already-scouted CdSe (`norris-bawendi-1996`), PbS (`kang-wise-1997`), and InP
(`almeida-2023`) routes, to reduce Quantum's dependence on a single
digitization-gated path. It documents the candidate locator, expected row
surface, rights/reuse posture, and direct-measurement status.

It transcribes no values, adds no `qd-*.yaml` row, runs no figure digitization,
edits no committed dataset, computes no baseline, and promotes no claim or
knowledge. It does **not** assert specific table contents from model memory — any
size/energy values remain unverified until a fetch-enabled source-artifact task
inspects the actual source.

## Why CdS, And Why This Candidate

The committed triage (`docs/reviews/quantum-open-direct-table-source-triage.md`,
`TASK-0347`) and `scout-02` (`TASK-0605`, Almeida InP) covered CdSe, PbS, InAs,
and InP. The source manifest also already records calibration/sizing-curve
sources (`yu-2003`, `moreels-2009`, `hens-moreels-2012`) and a model reference
(`brus-1984`). **CdS is not represented in any committed source entry.** A clean
CdS direct-measurement size series would diversify the campaign into a new
II-VI material family that is not a recolour of the existing CdSe rows.

The selected candidate is an early, widely cited CdS size-series characterization
study (size from structural methods, optical first-absorption feature reported
across a size range) rather than a fitted sizing-curve paper.

## Direct-Measurement vs Sizing-Polynomial Screen

Per the task's rejection rule, a candidate is admissible for **direct rows** only
if its values are measured per sample, not generated by evaluating a published
sizing polynomial:

- **Reject as rows** (calibration/polynomial): `yu-2003`, `moreels-2009`,
  `hens-moreels-2012` — these are fitted size↔energy curves; useful as calibration
  context, never as direct-measurement rows.
- **Reject** any value sourced from model memory or an effective-mass model
  (e.g. `brus-1984`) as a direct measurement.
- **Candidate (this scout):** Vossmeyer 1994 reports per-sample optical features
  against structurally measured sizes, so it is a direct-measurement *candidate* —
  **subject to** verifying that the committed value surface is a per-sample table
  (or accurately digitizable figure) and not a fitted curve.

## Decision

`PROMISING_NEW_FAMILY_DIRECT_TABLE_UNVERIFIED` — record the candidate; **preserve
a blocker**. Do not add a dataset row. The candidate cannot be promoted to an
admissible source until a future fetch-enabled source-artifact task does all of:

1. verifies a public/open or maintainer-provided access path for DOI
   `10.1021/j100082a044` (ACS subscription is not a committable path), and pins a
   checksum or records a maintainer-approved metadata-only source-copy policy;
2. confirms whether a printed per-sample **(size, absorption energy)** table
   exists, versus figure-only data requiring the
   `docs/quantum-direct-measurement-digitization-protocol.md` workflow;
3. confirms the values are direct per-sample measurements, not evaluations of a
   fitted sizing polynomial;
4. records per-point size/energy uncertainty semantics and direct-vs-derived
   labels, and decides whether the current quantum-dot schema represents CdS
   nanocluster sizing cleanly;
5. only then runs the row-readiness gate before any `qd-*.yaml` row is committed.

If access cannot be secured or the value surface turns out to be a sizing curve
or figure-only without admissible provenance, preserve a
`BLOCKER_SOURCE_ARTIFACT_NOT_COMMITTED` review (the `TASK-0334` pattern) rather
than relaxing the rules.
