## Decision: Option (a) — Make The Packaging Checks Deterministically Emitted

The two admissible resolutions were (a) make the publication-gate checks
deterministically emitted/replayed by the recorded command, or (b) move/relabel
them into a packaging section that Gate B intentionally ignores.

Option (b) was rejected on a hard schema constraint, not on preference. The
canonical result schema (`physics_lab/schemas/result.schema.json`) sets
`additionalProperties: false` at three relevant levels:

- the top-level result object (no new sibling packaging section is legal),
- the `verificationSummary` object (only `passed` and `checks` are allowed; no
  `packaging` sub-key), and
- each `verificationCheck` object (only `name`, `status`, `details`, `metrics`;
  no per-check `kind: gate_a_packaging` label).

A clean relabel would therefore require a schema change plus a Gate B comparator
change plus mutating the committed RESULT-0020 verification block — a broader,
cross-result change to comparator semantics that exceeds the minimal packaging
fix this task is scoped to.

Option (a) is minimal and schema-legal because all three checks are already
backed by data the recorded command computes deterministically:

- `zero_disagreement_ledger` ← `summary.total - summary.agree` and the
  disagreement item-id list (empty → the `"none"` scalar sentinel, matching the
  committed artifact; the verification metrics schema permits only scalar
  values, so the empty ledger is not an array).
- `frozen_input_checksum` ← the frozen fixture SHA-256 already recorded in
  `input_file_hashes.fixture.sha256`; `source_sha256_at_freeze` equals it
  because the pinned fixture *is* the freeze snapshot.
- `protected_result_not_rewritten` ← a deterministic publication-integrity
  assertion that this run writes new RESULT/RUN identities and does not rewrite
  the frozen RESULT-0007 / EXP-0006/RUN-0006 artifacts.

## Patch

A single minimal code patch, no metric change:

- `physics_lab/workflows/dimensional_validator.py` now emits the three
  provenance/packaging-integrity checks from run data for the exact RESULT-0020
  frozen live-74 publication replay, appended after the four pre-existing
  checks. The values are derived exactly so a fresh run reproduces the committed
  RESULT-0020 verification block without assigning RESULT-0020-specific
  packaging assertions to other dimensional-validator runs.

No schema change, no comparator change, and no edit to
`results/EXP-0006/RUN-0007/result.yaml` were required. The committed artifact is
byte-unchanged; its metrics, verdict, challenge-set rows, and `CLAIM-0005` are
untouched.
