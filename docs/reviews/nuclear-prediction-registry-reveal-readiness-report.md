## Reveal Gate State

All current actual entries must remain unscored before these gates pass:

- source preflight with reviewed source manifest and checksum policy;
- no-peek review against registry timing, task history, and committed datasets;
- explicit maintainer approval for the reveal task.

The summary lists every actual `PRED-*` entry under
`entries_must_not_be_scored_before_reveal_gate`. Newer entries that reference
the reveal protocol and source-readiness checklist still remain blocked because
those references are not a source manifest, no-peek result, or maintainer
approval. Older entries also carry missing-reference blockers for the newer
reveal protocol/checklist docs.

## Metadata Grouping

The summary groups entries by source task, target batch, and inferred model
family when the committed metadata supports it. These groupings are review
navigation aids only. They do not rank model quality, estimate source coverage,
or imply reveal eligibility.

Observed families include baseline/manual controls, pairing or odd-even
controls, shell or magic controls, neutron-excess/asymmetry controls,
factory coefficient transforms, feature-term selected entries, and the
shell-axis mini-wave. The groups reflect registry metadata strings and task
ids, not a measured scientific result.

## Wording Boundary

Registry volume is not scientific success. A higher `PRED-*` id or larger
entry count only means more frozen prospective records exist. It does not show
that any entry matched a later measurement, improved over a baseline, or is
eligible for claim promotion.

Allowed current wording:

- "registered prospective prediction registry entries";
- "awaiting source preflight and no-peek review";
- "not ready for reveal scoring";
- "deterministic registry metadata summary."

Forbidden current wording:

- "validated nuclear-mass predictions";
- "successful reveal";
- "confirmed model";
- "scientific success from registry volume";
- "blind prediction evidence" before no-peek review passes.

## Limitations

This report is a metadata audit. It does not inspect live sources, decide
future measurement eligibility, resolve partial-reveal coverage, or change any
frozen prediction. A future reveal task still needs the source manifest,
checksums, registry snapshot, no-peek audit, comparison command, scored and
excluded rows, limitations, and maintainer review required by the nuclear
prediction reveal protocol.
