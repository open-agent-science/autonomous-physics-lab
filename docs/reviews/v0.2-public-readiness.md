# v0.2 Public Readiness Review

## Scope

This review evaluates the current `v0.2` public-opening posture against
committed repository artifacts rather than planned work.

Reviewed surfaces:

- `README.md`
- `docs/status.md`
- `docs/mission-control.md`
- `docs/roadmap.md`
- `docs/public-release-gates.md`
- `docs/release-checklist.md`
- `docs/results/visual-summary.md`
- `docs/results/pendulum-gauntlet-100-summary.md`
- `docs/results/koide-tau-holdout.md`
- `docs/results/dimensional-analysis-validator-summary.md`
- `docs/multi-agent-dry-run.md`
- `docs/negative-results-registry.md`
- `docs/reproducibility-capsules.md`
- `docs/result-quality-rubric.md`
- `docs/releases/v0.2-public-alpha.md`

## Verdict

`PUBLIC AFTER NAMED BLOCKERS`

The repository now has enough reproducible scientific evidence and enough
contributor-workflow evidence to support a cautious public-facing `v0.2`
opening. It is not ready for immediate public release from the current branch
state because the remaining blockers are concrete release-governance issues,
not missing scientific ambition.

## Gate Review

### Gate 1 — Technical Stability

`PARTIAL`

Ready:

- canonical repository validation paths exist and are documented;
- strict validation mode exists;
- reproducibility capsules and core-reproduction helper now exist for the
  current flagship result surface;
- current worktree is clean on `main`.

Blockers:

- this review does not include a committed default-branch CI confirmation or a
  release-time validation log;
- `docs/multi-agent-dry-run.md` still contains a local `/private/tmp/...`
  path leak in its introductory links, which fails the gate's own hygiene
  standard.

### Gate 2 — Multi-Agent Contributor Pilot

`PASS`

Evidence present:

- `docs/multi-agent-dry-run.md` records at least three atomic task PRs;
- at least two contributors or agent sessions are represented;
- recorded PRs link back to canonical task ids;
- the note records CI and maintainer-review outcomes rather than only merge
  outcomes.

Remaining caution:

- the pilot is adequate as gate evidence, but contributor friction remains
  real for chat-first or weak-local-tooling environments.

### Gate 3 — Measurable Scientific Result

`PASS`

Evidence present:

- pendulum gauntlet result package with leaderboard, diagnostics, and limits;
- dimensional-analysis MVP benchmark with explicit frozen scope and known
  limitations;
- particle-mass track includes both scoped reproductions and strong negative
  results;
- negative-result surfaces are preserved as first-class repository memory.

Remaining caution:

- these are benchmark and falsification surfaces, not discovery-level physics;
- `EXP-0010` exists only as a guarded stress-test lane and should stay outside
  the flagship public-success set.

### Gate 4 — Public Narrative

`PARTIAL`

Ready:

- `README.md` now presents a result summary with explicit caveats;
- mission-control, status, and campaign pages align reasonably well with the
  current evidence surface;
- prepared `v0.2` release materials exist;
- overclaim language is mostly controlled.

Blockers:

- the current public-opening package still lacks an external-reviewer-oriented
  replication guide that tells a skeptical reviewer how to reproduce the main
  evidence surface without repository tribal knowledge;
- technical-hygiene issues such as the local path leak weaken confidence in
  the final public narrative review.

## What Is Ready

- a coherent pendulum benchmark surface;
- a falsification-first particle-mass campaign surface;
- a frozen dimensional-analysis MVP benchmark;
- visible negative results and reproducibility capsules;
- a branch-based contributor workflow with real pilot evidence;
- a prepared `v0.2` release-note and launch-pack surface.

## What Remains Uneven

- release-time technical evidence is not yet packaged into one explicit
  maintainer signoff artifact;
- external reviewer onboarding is weaker than internal contributor onboarding;
- some documentation still reflects maintainer-local workflow traces;
- result-quality signaling is stronger internally than it is in a compact
  public replication path.

## What Should Not Be Claimed

- APL has not solved physics.
- APL has not produced discovery-level particle-mass explanations.
- Pendulum benchmark success is not a proof of exact symbolic validity.
- Koide-track reproductions and falsifications do not settle all Koide-like
  ideas globally.
- `EXP-0010` is not a public success story.
- Thought-experiment consistency is not yet an implemented benchmark line.

## Named Blockers Before Public Opening

1. Remove maintainer-local path leaks from public-facing docs, starting with
   `docs/multi-agent-dry-run.md`.
2. Produce a compact external reviewer replication guide for the current major
   public evidence surface.
3. Record a release-time default-branch validation and CI signoff alongside
   the public-opening decision.

## Release Decision

Do not open the repository publicly from the current state.

Open it only after the named blockers above are resolved and the maintainer
re-runs the release checklist on the candidate public branch.
