# APL v0.2 Repository Opening Pack

## Purpose

This document is the maintainer-facing entrypoint for a future public opening
of Autonomous Physics Lab.

It does not declare that the repository should be opened today. It packages
the current repo-native materials that would support a cautious, verification-
first `v0.2` opening decision once the release gates are satisfied.

## What This Pack Is For

Use this pack when a maintainer wants one place to answer:

- what APL is trying to do;
- which benchmark and falsification surfaces are already real;
- which docs a new reader should open first;
- which release-facing artifacts are ready now;
- which claims still must remain out of bounds.

## Current Opening Narrative

APL is not a chatbot and not a discovery engine making broad physics claims on
demand.

The repo is strongest when described as:

- verification-first scientific infrastructure;
- a version-controlled memory for hypotheses, experiments, results, and tasks;
- a contributor-ready workflow for humans and coding agents;
- a small but real scientific campaign surface with both reproductions and
  falsifications.

The repo is not ready to be described as:

- a system that resolved physics;
- a theory-of-everything engine;
- a completed public scientific platform;
- a proof that Koide-like relations explain particle masses.

## Suggested Maintainer Opening Flow

1. Confirm that release gates are actually satisfied.
2. Re-run the repository validation path on a clean `main`.
3. Confirm the public default-branch history scan remains clean.
4. Review the current public wording surfaces for drift.
5. Use [docs/releases/v0.2-public-alpha.md](./releases/v0.2-public-alpha.md)
   as the canonical repo release-note draft.
6. If branch protection is unavailable while the repository is private because
   of the current GitHub plan, document the intended `main` protection policy,
   switch visibility only after the other gates pass, then enable protection
   immediately after the public switch and before broad announcement.
7. Point new contributors to [docs/use-your-agent.md](./use-your-agent.md)
   rather than relying on ad hoc onboarding messages.

## What Must Stay Explicit

- Benchmark results are scoped and limitation-bearing.
- Falsifications are first-class outputs, not embarrassing failures to hide.
- Claim promotion remains maintainer-reviewed.
- Public opening is downstream of release gates, not a substitute for them.

## Opening Risk Checklist

Before any public opening, confirm that none of the following drifted back in:

- "AI resolved physics"
- "discovery-level breakthrough"
- "proof"
- "globally valid"
- "Koide is true everywhere"
- "Koide is false everywhere"

If any of those framings reappear, the opening narrative is out of alignment
with the current repository evidence.
