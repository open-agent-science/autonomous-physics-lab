# v0.2-public-alpha

Prepared release notes for the public-facing `v0.2` opening of Autonomous
Physics Lab.

This repository-native draft can support a future GitHub release page,
maintainer summary, or release tag. The repository is already public; tagging
or broad release promotion remains a separate maintainer decision.

## Included

- fourteen canonical experiments across multiple public-facing benchmark and
  validation surfaces;
- a visible result layer with benchmark summaries, figures, and stored
  limitations;
- a negative-results registry so falsifications remain first-class outputs;
- contributor workflow docs for humans and coding agents;
- canonical task files, active-board sync, and maintainer review helpers;
- repository-wide validation and strict validation support.

## Scientific Scope

This release remains intentionally narrow and verification-first.

It presents:

1. a classical benchmark surface built around pendulum and damped-oscillator
   verification;
2. a particle-mass campaign with scoped charged-lepton reproduction, tau
   holdout, neutrino falsification, and quark falsification surfaces;
3. a dimensional-analysis validator MVP benchmark;
4. a Nuclear Mass Surface validation track with `EXP-0012` baseline evidence,
   sandbox-only autonomous pilot evidence, and retrospective post-AME2020
   guardrails.

Separate from that highlight set, the repository also contains `EXP-0010`, a
guarded muon g-2 formula-search stress test that remains outside the public
success surface.

It does not present:

- discovery-level physics claims;
- a theory of particle masses;
- proof that Koide-like structure is globally true or globally false;
- an autonomous scientific review system that bypasses human judgment.

## Contributor Surface

The release-facing contributor story is:

- branch-based work only;
- canonical `TASK-*` files and `docs/task-views/*.md` as the shared coordination
  layer;
- coding agents are welcome, but they follow the same verification-first and
  review-first rules as humans;
- maintainers review claims, wording, and merge decisions.

Start here:

- [docs/use-your-agent.md](../use-your-agent.md)
- [docs/agent-work-menu.md](../agent-work-menu.md)

## Release Strengths

- current evidence is easier to scan than before;
- both reproductions and falsifications are visible in the same repo;
- contributor onboarding is clearer and more repo-native;
- overclaim wording has already been audited and tightened.

## Release Limits

- the repository is public-alpha live, but broad release/tag promotion remains
  gated, not automatic;
- some campaign surfaces are still intentionally planning-first;
- `EXP-0010` remains a numerology-risk stress test rather than a release
  highlight;
- Nuclear Mass Surface remains validation evidence only: post-AME2020
  evaluation is retrospective and no candidate is promoted automatically;
- contributor scale, not just contributor correctness, is still limited.

## What Not To Claim

Do not present `v0.2-public-alpha` as:

- AI-discovered physics;
- a global particle-mass explanation;
- a finished scientific platform;
- an engine that no longer needs maintainer review discipline.

Do present it as:

- a verification-first repository opening;
- a small but real scientific campaign surface;
- a reproducible, reviewable public scientific memory prototype;
- an invitation for careful contributors and coding agents to help extend the
  benchmark layer.
