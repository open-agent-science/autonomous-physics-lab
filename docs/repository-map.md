# Repository Map

Use this map when the repository root feels busy. APL keeps scientific memory
as version-controlled files, so the top level intentionally exposes several
artifact classes. Not every visible directory is an active work lane.

## Core Runtime And Automation

| Path | Role |
| --- | --- |
| `physics_lab/` | Python package: CLI, engines, datasets, workflows, registry loaders, validation, review helpers. |
| `scripts/` | Maintainer and agent helpers for mission selection, review, closeout, snapshots, source ingestion, and campaign runs. |
| `tests/` | Fast deterministic tests and repository-integrity checks. |
| `.github/` | CI and post-merge automation. |
| `examples/` | Reproducible workflow configs and smoke-test examples. |

## Current Work Coordination

| Path | Role |
| --- | --- |
| `tasks/` | Canonical task contracts. Agents choose `READY` tasks; DONE history stays here until a separate archive migration is proven safe. |
| `docs/task-views/` | Generated human navigation over current work. Agents should query `apl_mission.py` or these views, not a committed static task index. |
| `missions/` | Mission-control policy and current campaign recommendation data. |
| `agents/` | Machine-readable role profiles for review, architecture, scientific direction, and other agent roles. |
| `campaign_profiles/` | Per-campaign autonomy, safety, capacity, and portfolio source fields. Edit profiles, then regenerate `_catalog.yaml`. |
| `campaign_profiles/_catalog.yaml` | Generated campaign portfolio index colocated with the source profiles; service-style name marks it as generated aggregate state, not an editable campaign profile. |

## Scientific Memory

| Path | Role |
| --- | --- |
| `hypotheses/` | Reviewed hypothesis artifacts. |
| `hypothesis_proposals/` | Campaign-scoped unreviewed hypothesis proposals. |
| `experiments/` | Reviewed experiment definitions. |
| `experiment_proposals/` | Campaign-scoped experiment proposal artifacts. |
| `results/` | Canonical result packages and review-tier artifacts. |
| `claims/` | Claim records; claim promotion remains review-gated. |
| `knowledge/` | Reusable reviewed knowledge notes. |
| `prediction_registry/` | Pre-registered prediction artifacts awaiting reveal or scoring. |
| `agent_runs/` | Sandbox or task-scoped evidence bundles from autonomous agents. |
| `microtask_runs/` | Completed scientific microtask records. |

## Data And Templates

| Path | Role |
| --- | --- |
| `data/` | Curated datasets, manifests, source artifacts, checksums, and snapshot plans. |
| `templates/` | Reusable source, result, extraction, and artifact templates. |

## Narrow Or Legacy Surfaces

| Path | Lifecycle |
| --- | --- |
| `approximation_probes/` | Retained narrow deterministic approximation-breakdown track with live code and tests. Do not expand into a broad formula-audit registry. |
| `docs/legacy/constants-verification/` | Archived legacy constants-verification worked example. Do not recreate `constants_verification/` at root by default. |
| `docs/legacy/hypothesis-register/` | Archived legacy hypothesis-register pilot. New ideas go to `hypothesis_proposals/` or reviewed `hypotheses/`. |

## Local Or Generated Checkout Artifacts

These may appear in a local checkout but are not canonical repository
architecture:

| Path | Meaning |
| --- | --- |
| `_snapshots/` | Maintainer/strategy handoff snapshots generated by `./scripts/apl_snapshot.sh`. |
| `.worktrees/`, `_worktrees/`, `.claude/worktrees/` | Local agent worktrees. Do not commit. |
| `.pytest_cache/`, `.pytest-basetemp/`, `.ruff_cache/`, `.DS_Store` | Local tool or OS cache files. |

## Migration Rule

Broad path moves must happen through a dedicated task that updates loaders,
validators, docs links, snapshots, tests, and agent instructions together. Do
not introduce committed static files solely for volatile agent routing; agents
should use dynamic scripts, mission output, and generated human task views.
