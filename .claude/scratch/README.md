# Claude Agent Scratch Directory

This directory is the **Claude Code** scratch location for throwaway
artifacts only:

- PR-body scaffolds produced by `scripts/apl_task_pr_helper.py scaffold`
- short helper notes generated by a Claude Code session during a task
  lifecycle
- any temporary file a Claude Code agent needs to write inside the
  project

The `.claude/` namespace is Claude Code-scoped. Codex and other
non-Claude agents should use their own native scratch convention (for
Codex, the per-session worktree under `~/.codex/worktrees/`) rather
than writing here.

## Rules

- **Contents are gitignored.** Only this `README.md` and `.gitkeep` are
  tracked. Anything else you write here will not be committed.
- **Not an audit trail.** Do not use this directory in place of
  `_snapshots/`. `_snapshots/` holds review bundles produced by
  `./scripts/apl_review_bundle.sh` and is the audit-trail surface
  expected by maintainer review.
- **Not a publication surface.** Nothing here is reviewed, validated,
  or part of the scientific memory. Results, predictions, claims, and
  knowledge entries must go to their canonical destinations under
  `results/`, `prediction_registry/`, `claims/`, or `knowledge/`.
- **Safe to delete at any time.** Agents and contributors may clear
  the directory between tasks without losing reviewable state.

## Canonical filenames used by helpers

Common scratch artifacts agents and helpers write here:

- `apl-pr-body.md` / `apl-pr-body-<task-id>.md` — task PR body
  scaffolds produced by `scripts/apl_task_pr_helper.py`.
- `apl-closeout-pr-body.md` — closeout PR body produced by
  `scripts/apl_closeout_pr_helper.py`.
- `apl-microtask-pr-body.md` — microtask PR body produced by
  `scripts/apl_microtask_pr_helper.py`.

The directory is registered in `.claude/settings.json` via
`permissions.allow` (`Read|Write|Edit(.claude/scratch/**)`) so writes
do not prompt the user.

## Portability

The path `.claude/scratch/` is **relative to the repo root** and uses
forward slashes, so it is portable across macOS, Linux, WSL, Git Bash,
and plain Windows shells where Claude Code runs. Claude Code sessions
should not substitute `/tmp/`, `%TEMP%`, `%TMP%`, or any absolute
platform-specific scratch path.

The `permissions.additionalDirectories: ["/tmp", "/private/tmp"]`
entry in `.claude/settings.json` is a Unix-only fallback that
preserves backward compatibility with older Claude Code sessions that
still write to `/tmp/`. On Windows that entry is a no-op and this
directory becomes the only correct Claude scratch location.

See `CLAUDE.md` "Agent Scratch Directory" and
`docs/agent-task-protocol.md` for the canonical references.

## Why this exists

Agents previously wrote PR-body scaffolds to `/tmp/apl-pr-body-*.md`,
which triggered Claude Code "Path is outside allowed working
directories" permission prompts even when the path matched the
`Write(/tmp/apl-*)` allowlist entry, because the workspace boundary is
a separate setting (`permissions.additionalDirectories`).
Routing those artifacts inside the project removes the prompt and
keeps them visible to the same agent across follow-up turns.
